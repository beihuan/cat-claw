# 汉语言文学学术文献检索与综述工具

## 概述

本技能专为汉语言文学专业设计，聚焦两个高频学术场景：

1. **方向综述模式**：围绕一个研究方向（如"《红楼梦》叙事学研究"），系统检索该方向已有的学术成果，并自动起草一份结构化的文献综述。
2. **论点溯源模式**：用户给出一句论述或一个理论（如"巴赫金的对话理论在中国古典小说分析中的引入始于90年代"），帮助其找到最可能的文献出处作为引用支撑。

## ⛔ 反幻觉铁律（最高优先级）

> 以下规则的优先级高于本文件中的所有其他指令。任何情况下都不得违反。

1. **绝不编造文献**：不得虚构任何论文标题、作者姓名、期刊名称、年份、卷期页码或 DOI。如果检索不到结果，直接告知用户"未检索到相关文献"，而不是猜测或编造。
2. **绝不编造引用关系**：不得声称"某某学者在某某论文中提出过这个观点"，除非确实在检索结果中找到了该文献中包含相关内容的证据。
3. **不确定即标注**：如果对某条文献信息的准确性存疑（例如年份记不准、作者名可能有误），必须用 `[待核实]` 标签显式标注，并建议用户自行到数据库核对。
4. **区分检索结果与推理**：明确区分"从数据库中检索到的文献"和"基于知识推理认为可能相关的文献"。后者必须单独列出并标注 `[基于模型知识，非实时检索，请用户自行核实]`。
5. **宁缺毋滥**：如果只找到 3 篇可靠文献，就只返回 3 篇。绝不为凑数量而降低可信度。
6. **出处链接必附**：每一条引用的文献**必须附带可访问的出处链接**（URL）。链接来源优先级：知网页面 > 万方/维普页面 > 国家哲学社会科学文献中心页面 > 期刊官网 > DOI 链接 > 其他可验证来源。如果某条文献来自模型知识且无法提供链接，必须标注 `[无链接-请用户自行核实]` 并归入单独的「待核实文献」分区。**绝不编造链接**——没有检索到 URL 的文献不得凭空生成一个看似合理的链接。

## 工作流 A：方向综述模式

**触发条件**：用户描述了一个研究方向/主题，并要求了解已有研究成果或生成文献综述。

### A1. 明确检索范围

向用户确认以下信息（如未提供则主动追问）：

| 信息项 | 说明 | 默认值 |
|--------|------|--------|
| **研究方向** | 尽可能具体，如"鲁迅小说的讽刺叙事研究"而非"鲁迅研究" | 无，必须提供 |
| **时间范围** | 如"近十年"、"2015年至今" | 不限 |
| **论文数量** | 期望综述覆盖多少篇 | 15-25 篇 |
| **综述目的** | 学位论文绪论？课程作业？开题报告？ | 学位论文绪论 |
| **语种偏好** | 以中文为主是否也需要外文文献 | 以中文为主 |
| **细化角度** | 是否聚焦某个理论视角或方法论 | 无特定限制 |

**追问示例**：
> "您提到想查'《红楼梦》的叙事研究'。这个方向较宽泛，请问您是否聚焦于某个具体角度，比如叙事视角、叙事时间、空间叙事、或不可靠叙述者？另外，您更关注近年的研究动态还是需要追溯到早期经典论述？"

### A2. 制定搜索策略

1. **生成查询关键词组**：围绕用户主题生成 3-5 组关键词变体：
   - 核心词（如"红楼梦 叙事"）
   - 同义/近义替换（如"红楼梦 叙述学"、"红楼梦 叙事结构"）
   - 理论框架词（如"红楼梦 热奈特"、"红楼梦 叙述视角"）
   - 扩大范围词（如"古典小说 叙事"、"明清小说 叙事学"）

2. **确定数据源优先级**（见下方数据源表）。

3. **设定迭代上限**：每个数据源最多 3 轮不同关键词组合的搜索。

### A3. 执行多源检索

**数据源优先级表**（中文文学方向）：

| 优先级 | 数据源 | 说明 | 访问方式 |
|--------|--------|------|----------|
| 0 | **秘塔AI搜索 (Metaso)** | AI 搜索引擎，`metaso_web_search` 做通用搜索，`metaso_chat` 做智能问答辅助，`metaso_web_reader` 抓全文 | MCP HTTP 调用（API Key 存于长期记忆） |
| 1 | **web-search (z-ai-web-dev-sdk)** | 通用网页搜索，适合用 `site:` 操作符定向搜索学术网站 | SDK 内置 |
| 2 | **中国知网 (CNKI)** | 汉语言文学最核心数据库，覆盖期刊、学位论文、会议 | 需要用户提供登录环境或导出数据 |
| 3 | **万方数据 / 维普 (VIP)** | 期刊与学位论文补充 | 同上 |
| 4 | **国家哲学社会科学文献中心** | 开放获取，适合社科人文 | 公开访问 |
| 5 | **Semantic Scholar** | 部分中文文献有收录，外文文献补充 | 公开 API |

**关键访问约束**：
- 知网、万方、维普均为需要登录的商业数据库。如果当前环境**无法直接访问**，必须：
  1. 明确告知用户"我无法直接登录知网检索"
  2. 请求用户在知网中搜索指定关键词并将结果复制给我
  3. 或者请用户导出文献列表（如 EndNote/NoteExpress 格式、Excel 或纯文本）
- **绝不假装自己访问了知网并返回编造的结果**

**检索执行策略**（多轮迭代）：

**第一轮：秘塔 web_search + web-search 并行搜索**
- 秘塔 `metaso_web_search`：使用 `scope: "webpage"`（默认），**不要用 `scope: "paper"`**（实测与 webpage 返回完全相同结果，scope 参数未生效）。设置 `includeSummary: true`
- web-search：用 `site:` 操作符定向搜索学术域名，如：
  - `"关键词 site:cssn.cn"` — 中国社会科学网
  - `"关键词 site:ncpssd.cn"` — 国家哲学社科文献中心
  - `"关键词 site:cnki.net"` — 知网公开页面
  - `"关键词 site:cqvip.com"` — 维普
  - `"关键词 filetype:pdf"` — 开放获取 PDF
- 每轮生成 3-5 组关键词变体（核心词、同义词、理论框架词、扩大范围词）

**第二轮：秘塔 metaso_chat 智能问答辅助**
- 用 `metaso_chat` 提问："请列出关于[研究主题]的CSSCI核心期刊论文，包含标题、作者、期刊、年份"
- metaso_chat 返回的 citations 中可能包含 web_search 未覆盖的来源
- **注意**：metaso_chat 的回答本身不可直接引用（是 AI 生成），但其 citations 中的链接可以作为线索去抓取原文

**第三轮：针对性补充**
- 根据前两轮结果中的缺口，用 web-search 补充搜索特定子主题
- 如发现某篇重要论文只有标题，尝试用 `metaso_web_reader` 或 `web-reader` 抓取其页面

**每轮搜索后立即执行后过滤**（见下方），不要等所有搜索完成再过滤。

**⚠️ 秘塔搜索后过滤规则（实测必要）**：
秘塔搜索结果中非学术来源噪声极大（实测 qidian.com、baidu.com、bilibili.com、douban.com 等占比超 50%），必须强制过滤：

```python
# 必须排除的域名（非学术来源）
BLOCK_DOMAINS = [
    "qidian.com", "baike.baidu.com", "bilibili.com", "douban.com",
    "zhihuishu.com", "douyin.com", "sina.com.cn", "sohu.com",
    "qq.com", "weibo.com", "jd.com", "taobao.com", "zikaosw.cn",
    "kemingbaike.com", "xuges.com",
]

# 优先保留的学术域名
ACADEMIC_DOMAINS = [
    "cnki.net", "ncpssd.cn", "cssn.cn", "wanfangdata.com.cn",
    "cqvip.com", "sinoss.net", "core.ac.uk", "pku.edu.cn",
    "tsinghua.edu.cn", "fudan.edu.cn", "nju.edu.cn", "ruc.edu.cn",
    "bnu.edu.cn", "cass.cn", "ucass.edu.cn", "wxg.org.cn",
    "bisu.edu.cn", "hanspub.org", "researchcommons.org",
    "artsbj.cn", "wenshuyi.com", "qikan.com.cn",
    "etdlib.bnu.edu.cn", "huanghuai.edu.cn",
]
```

### A3.5. 全文获取与精读

> **本步骤是保证综述质量的关键环节。** 未经全文阅读的文献，综述中不得引用其具体观点（反幻觉铁律第2条）。

**全文获取策略**（按优先级逐层尝试）：

**第一层：秘塔 web_reader**
- 使用 `metaso_web_reader` 读取论文页面，参数 `format: "markdown"`
- 适用于大多数可公开访问的网页版论文
- 注意：部分网站返回内容极少（如 qikan.com.cn 可能只返回标题），需判断是否有效

**第二层：z-ai web-reader (page_reader)**
- 使用 `z-ai function -n page_reader` 读取网页
- 适用于秘塔 web_reader 抓取失败或内容不完整的页面

**第三层：PDF 直接下载 + pymupdf 提取**
- 对可直接下载的 PDF 链接，用 `curl -L -o` 下载
- 用 `pymupdf`（`fitz`）提取全文：`pip install pymupdf --break-system-packages`
- 适用于：大学学报 PDF、开放获取期刊 PDF、机构库 PDF

**第四层：无头浏览器 (agent-browser)**
- 使用 agent-browser 访问需要交互的网站
- 适用于：需要点击"下载PDF"按钮、需要登录才能访问全文的场景

**第五层：用户协助（兜底）**
- 用户有知网/万方账号，可导出或复制全文内容
- 用户截图或粘贴关键段落

**全文有效性判断**：
- 有效全文：≥ 1000 字，包含摘要、正文、参考文献等结构
- 仅摘要/标题：< 1000 字，或只有元数据无正文
- 无效：空内容、验证码页面、付费墙

**精读与标注**：
- 对成功获取全文的文献，提取并记录：核心论点、研究方法、论证逻辑、关键结论、与用户研究方向的关联度
- 对仅获取到标题/摘要的文献，标注 `[仅标题/摘要，内容未经确认]`
- **综述中对未读全文的文献只能做背景提及（如"学界对此已有讨论"），不得引用具体观点或声称"某某学者提出了某某观点"**

**输出**：生成一份「文献精读笔记」，每篇文献包含：
```
[序号] 作者. 标题[J]. 期刊, 年份. [链接](URL)
- 全文状态：✅ 已读全文 / ⚠️ 仅标题摘要 / ❌ 无法获取
- 核心论点：（已读全文时填写）
- 研究方法：（已读全文时填写）
- 与选题关联度：高/中/低
- 备注：（待核实项、存疑项等）
```

### A4. 整理与去重

1. **去重**：按标题相似度 ≥90%（去除标点后比对）或 DOI 一致性判断重复。
2. **筛选排序**：
   - 第一级：与用户主题的相关性
   - 第二级：发表层次（CSSCI/核心期刊 > 一般期刊 > 学位论文）
   - 第三级：被引次数（如有数据）
   - 第四级：时间新旧（近年优先）
3. **去粗取精**：剔除明显偏离主题的结果，保留目标数量的论文。

### A5. 生成文献综述

根据检索到的**真实文献**，按以下结构起草综述：

```
## [研究方向] 文献综述

### 一、研究概况
简述该方向的研究起始时间、总体发展脉络、研究热度趋势。

### 二、主要研究进展
按主题/视角分类梳理已有研究，每个分支包括：
- 代表性学者及其核心观点
- 关键论文及其贡献
- 不同学者之间的观点异同与对话

### 三、研究方法与理论视角
概述该方向主要采用的研究方法和理论框架。

### 四、现有研究的不足与未来展望
指出当前研究中的空白、争议或薄弱环节，提出可能的研究方向。

### 参考文献
[按 GB/T 7714-2015 格式列出所有引用的文献，每条附带出处链接]

#### 待核实文献（基于模型知识，无检索链接）
[如有，单独列出无法提供 URL 的文献，标注 [无链接-请用户自行核实]]
```

**综述写作铁则**：
- 综述中提及的每一篇文献**必须出现在参考文献列表中**
- 综述中描述的每一个观点**必须注明出处**（作者+年份），并在正文中以脚注或行内链接形式附上该文献的 URL
- 如果某段综述内容基于模型知识而非检索结果，必须标注 `[此处论述基于模型知识，所引文献待用户核实]`
- 参考文献格式统一采用 **GB/T 7714-2015**（中国国家标准），每条文献**必须附带出处链接**

### A6. 交付与后续

交付综述初稿后，主动提供：

- 📋 **文献清单导出**："需要将参考文献导出为 NoteExpress/EndNote/BibTeX 格式吗？"
- 🔍 **深入某一分支**："需要对综述中某个子主题做更深入的文献挖掘吗？"
- ✏️ **修改调整**："需要调整综述的篇幅、结构或重点覆盖方向吗？"
- 📊 **研究趋势图**："需要梳理该方向的年度发文量变化趋势吗？"

## 工作流 B：论点溯源模式

**触发条件**：用户给出了一句具体的论述、一个理论观点或一个学术判断，希望找到可引用的出处。

### B1. 理解待溯源的论点

解析用户给出的论述，拆分为可检索要素：

| 分析维度 | 示例 |
|----------|------|
| **核心观点** | "巴赫金的对话理论被引入中国古典小说研究始于90年代" |
| **涉及的学者/理论** | 巴赫金、对话理论（dialogism） |
| **涉及的研究对象** | 中国古典小说 |
| **涉及的时间断言** | 90年代 |
| **论点类型** | 时间断言 / 观点归属 / 方法论判断 / 事实陈述 |

**追问示例**（如论点不够清晰）：
> "您提到'鲁迅的反讽策略受到了果戈理的影响'。请问这是您自己的观点需要找支撑文献，还是您在某处读到过但忘记了出处？这会影响我的搜索策略。"

### B2. 构建搜索策略

根据论点类型差异化搜索：

**类型一：观点归属型**（"某学者提出了某观点"）
- 直接搜索该学者的论文/著作
- 搜索引用该学者的综述文章

**类型二：时间/史实断言型**（"某现象始于某时期"）
- 搜索该现象的研究综述/学术史梳理文章
- 搜索该时期的相关原始文献

**类型三：方法论判断型**（"某理论适用于某对象"）
- 搜索该理论+该对象的交叉文献
- 搜索方法论反思类的文章

**类型四：事实陈述型**（"某作品具有某特征"）
- 搜索该作品的专门研究文献
- 搜索讨论该特征的文献

### B3. 执行检索

- 数据源优先级同工作流 A（知网优先）
- 每组关键词至少搜索 2 个数据源
- 特别留意综述类文章和学术史类文章（这类文章常常包含对已有观点的梳理和归属说明）

### B4. 匹配与验证

对每个检索结果，评估其与用户论点的匹配程度：

| 匹配等级 | 说明 | 标注 |
|----------|------|------|
| ⭐⭐⭐ 直接出处 | 论文中明确包含了与用户论点高度一致的表述 | `[强匹配-可直接引用]` |
| ⭐⭐ 支撑性引用 | 论文内容在方向上支撑用户论点但表述不完全一致 | `[中匹配-可作为支撑引用]` |
| ⭐ 相关参考 | 论文与论点涉及同一主题但未直接涉及该具体观点 | `[弱匹配-可作为背景参考]` |

### B5. 返回溯源结果

**输出格式**：
```
## 论点溯源结果

### 待溯源论点
> "[用户原始论述]"

### 检索到的可能出处

#### ⭐⭐⭐ 直接出处
1. 作者. 标题[J]. 期刊名, 年, 卷(期): 页码. [链接](URL)
   - 匹配理由：该论文第X页明确提出"……"
   - 来源数据库：知网/万方/…

#### ⭐⭐ 支撑性引用
1. 作者. 标题[J]. 期刊名, 年, 卷(期): 页码. [链接](URL)

#### ⭐ 相关参考
```

## 故障排除

### 常见问题与处理

| 问题 | 处理方式 |
|------|----------|
| 无法访问知网/万方/维普 | 明确告知用户无法直接访问，请求其手动搜索后将结果粘贴给你。绝不假装已访问过。 |
| 搜索结果为零 | 尝试扩展关键词（去除限定词、使用上位概念），如仍无结果，坦诚告知"该方向可能文献较少或关键词需要调整"。 |
| 用户论点涉及的理论/学者在模型知识中不确定 | 标注 `[待核实]`，告知用户"我对此信息的准确性不确定，建议您核实原始文献"。 |
| 中外文文献混合需求 | 先完成中文检索，再补充外文部分。外文部分通过 Semantic Scholar 检索。 |
| 找到疑似相关的论文但无法确认其内容 | 列出该论文的基本信息，标注 `[标题相关，内容未经确认，请用户自行查阅全文核实]`。 |
| 用户提供的知网导出格式不规范 | 尝试解析，如果无法准确解析，请求用户重新导出为 Excel 或纯文本格式。 |

### 优雅降级

如果主要数据源（知网、万方）均不可用：
1. 用 Semantic Scholar 和国家哲学社会科学文献中心检索
2. 基于模型自身的学术知识提供参考方向（但必须全部标注 `[基于模型知识，非实时检索，请用户自行核实]`）
3. 给出具体的知网搜索建议（关键词+筛选条件），让用户自行检索

> "⚠️ 由于当前环境无法直接访问中文学术数据库，以下文献信息基于模型知识而非实时检索结果，请务必到知网等数据库逐一核实后再引用。我同时为您准备了具体的检索建议，方便您自行查找。"

## 输出格式标准

### 参考文献格式

统一采用 **GB/T 7714-2015**（中国国家标准引文格式），**每条文献必须附带出处链接**：

**期刊论文：**
```
[序号] 作者. 题名[J]. 刊名, 出版年, 卷(期): 起止页码. [链接](URL)
示例：[1] 陈平原. 中国小说叙事模式的转变[J]. 文学评论, 1988, (3): 65-78. [链接](https://kns.cnki.net/...)
```

**学位论文：**
```
[序号] 作者. 题名[D]. 授予单位所在地: 授予单位, 年份. [链接](URL)
示例：[1] 张三. 《红楼梦》叙事视角研究[D]. 北京: 北京大学, 2020. [链接](https://kns.cnki.net/...)
```

**专著：**
```
[序号] 作者. 书名[M]. 出版地: 出版社, 年份. [链接](URL)
示例：[1] 申丹. 叙述学与小说文体学研究[M]. 北京: 北京大学出版社, 2004. [链接](https://book.douban.com/...)
```

**论文集中析出文献：**
```
[序号] 析出文献作者. 析出文献题名[C]// 论文集编者. 论文集题名. 出版地: 出版社, 年: 页码. [链接](URL)
```

**无链接文献（仅限模型知识来源）：**
```
[序号] 作者. 题名[文献类型]. 刊名/出版社, 年份. [无链接-请用户自行核实]
```

## 触发示例

### ✅ 应当触发

**方向综述模式（工作流 A）：**
- "帮我查一下'鲁迅小说讽刺艺术'方面有哪些已有研究"
- "我想了解'《诗经》比兴手法'的研究现状，能生成一个文献综述吗"
- "近十年关于'网络文学经典化'有哪些重要论文"
- "帮我梳理一下'巴赫金对话理论在中国文学批评中的应用'的研究脉络"
- "我的开题方向是'明清才子佳人小说的性别叙事'，帮我做文献综述"

**论点溯源模式（工作流 B）：**
- "我写了'《红楼梦》的叙事时间受到了《源氏物语》的影响'，帮我找支撑这个论点的参考文献"
- "'现代汉语的欧化语法现象始于五四时期'——这个说法的出处在哪"
- "谁最早提出了'中国叙事学'这个概念？帮我找一下"
- "我论文里提到'意象批评是中国古代文论的核心方法之一'，需要一个引用出处"
- "'鲁迅的《故事新编》体现了后现代主义的解构特征'——有哪些论文讨论过类似观点？"

### ❌ 不应触发
- "帮我写一篇关于鲁迅的论文" → 写论文正文非本技能职责
- "翻译一下这段古文" → 翻译非本技能职责
- "鲁迅是哪年出生的" → 百科问答非本技能职责
- "推荐几本好看的小说" → 书籍推荐非本技能职责
- "帮我润色这段话的表达" → 文本润色非本技能职责
